Manual de salga¶

SALGA significa System for Automated Learning based in Genetics Algorithms, y es un programa desarrollado en python para el simular la evolucin por seleccin natural de los individuos de una poblacin de soluciones a un problema. Est orientado tanto a docencia como a desarrollo de proyectos en este campo.

Primeros pasos¶

Instalación¶

Descomprimir el archivo salga-xxx.zip.

El ordenador deber tener instalado python 3.x. Para algunos ejemplos tambin deber estar instalado el mdulo matplotlib. En algunas distribuciones de linux puede ser necesario instalar adems el paquete python-tk.

Ejecutar el programa Salga¶

En windows, ejecutar salga.pyc con doble click. En MacOSX y en linux, abrir un terminal y teclear python salga.pyc.

Definición del fitness, alfabeto y tipo de GA¶

Por su propia naturaleza, los Algoritmos Genticos requieren, para ser adaptados a un problema concreto, que se defina una funcin de fitness (o funcin de calidad) que indica cmo de bueno es un individuo particular de cara a resolver un determinado problema. ste fitness es la parte dependiente del dominio y debe programarse previamente para que el sistema pueda aplicarse al problema concreto.

Adicionalmente puede definirse una funcin phenotype que permita obtener una representacin del individuo a evaluar a partir del cromosoma que lo describe. Por otro lado debemos seleccionar el tipo de algoritmo gentico a utilizar (classic, floating o permutation) e indicar el alfabeto a utilizar en el problema.

Sobre el tipo de Algoritmo Genético¶

El tipo de Algoritmo Gentico a utilizar puede ser:

  • Classic
    • El alfabeto es un conjunto finito de smbolos.
    • La mutacin se realiza a nivel de gen, eligiendo un nuevo valor del alfabeto.
    • El emparejamiento se realiza cortando las cadenas de los padres en un punto aleatorio y pegndolas cruzadas.
  • Floating
    • El alfabeto se forma con todos los posibles valores flotantes en un intervalo [a,b] definido en el campo alfabeto.
    • La mutacin se realiza un salto aleatorio desde el valor actual con distribucin gausiana y temple simulado.
    • El emparejamiento se realiza igual que en el modo classic.
  • Permutation
    • El alfabeto es un conjunto finito de smbolos.
    • Un cromosoma se forma por una permutacin concreta del alfabeto, es decir, la longitud del cromosoma es la longitud del alfabeto.
    • La mutacin consiste en intercambiar dos genes de posicin.
    • El emparejamiento se construye copiando una subsecuencia del padre A y rellenando el resto con los valores que no estn, tomndolos del padre B y manteniendo el orden parcial, aunque no la posicin.
Sobre el alfabeto¶

Dependiendo del tipo de Algoritmo Gentico elegido, el alfabeto se especifica de diferente modo:

  • En classic, el alfabeto se representa por una lista de smbolos separados por comas y entre corchetes. Ejemplo: [0, 1] alfabeto binario.
  • En floating, el alfabeto se especifica por el par de valores que definen el intervalo, entre corchetes. Ejemplo: [0, 10] todos los valores flotantes en el intervalo [0, 10].
  • En permutation, el alfabeto se representa por una lista de smbolos separados por comas y entre corchetes, igual que en el modo classic. Ejemplo: [1, 2, 3, 4, 5] permutaciones de los nmeros naturales del 1 al 5.
Sobre el fitness¶

La definicin del fitness se realizar en un archivo python independiente que se cargar posteriormente desde el programa salga. Veamos un ejemplo sencillo cuyo objetivo es encontrar el nmero x tal que x^2 sea 488670902500:

import math
target = 488670902500

def phenotype (chromosome): # computes the decimal number represented by the chromosome
    res = 0
    l = len(chromosome)
    for v in range(l):
        if chromosome[l-v-1] != 0: res += 2**v
            return res

def fitness (chromosome): # aproximates x^2 = target 
    fen = phenotype(chromosome)
    return 1.0 / (1.0 + abs(target - fen**2)) # maximum value when fen**2

Es este cdigo, la funcin phenotype recibe un cromosoma y devuelve el nmero decimal obtenido a partir del genotipo, es decir, transforma, por ejemplo, 1 0 0 1 en el valor decimal 5. Si no se especifica, el fenotipo es el propio cromosoma.

La funcin fitness devuelve un valor tanto ms alto cuanto ms cercano es el nmero representado por un cromosoma a nuestro objetivo. Se ha construido adems de modo que su mximo valor posible es 1.0, lo cual es bastante conveniente para la representacin grfica y para detener automticamente el proceso evolutivo cuando el fitness del mejor individuo sea 1.

Carga de parámetros¶

Es posible cargar valores para los parmetros cuando se realiza la carga del fitness, tales como el alfabeto, las probabilidades de mutacin y emparejamiento, la longitud del cromosoma y el mtodo a utilizar, entre otros.

La carga de parmetros desde el archivo de fitness se realiza mediante un diccionario llamada 'parameters'. Las claves del diccionarios pueden tomar los valores siguientes:

  • 'alphabet': el alfabeto a usar. Tipo: lista.
  • 'type': tipo de gentico a utilizar. Tipo: string
  • 'elitism': Si se activa elitismo. Tipo: boolean.
  • 'memetic': Si se usa memtico. Tipo: boolen.
  • 'norm': Si se usa normalizacin. Tipo: boolean.
  • 'unique': Si se fuerza a que cada individuo sea nico. Tipo: boolean.
  • 'normfactor': Factor de normalizacin. Tipo: float.
  • 'pmut': Probabilidad de mutacin. Tipo: float.
  • 'pcross': Probabilidad de emparejamiento. Tipo: float.
  • 'chromsize': Tamao del cromosoma. Tipo: int.
  • 'popsize: Tamao de la poblacin. Tipo: int.
  • 'target': fitness objetivo. Tipo: float.
  • 'tournament': Tamao de torneo. Tipo: float.
  • 'seed': Individuo semilla para evolucin 'seed'. Tipo: lista.

Controles¶
Controles¶
  1. Seleccin del tipo de genticoa utilizar puede ser classic, floating o permutation. Cada tipo tiene sus propia interpretacin del alfabeto y sus propios operadores de emparejamiento y mutacin.
  2. Load Fitness Function carga el cdigo python que define el fitness y en su caso el phenotype.
  3. Init crea la poblacin inicial con valores aleatorios.

  6. Pcross probabilidad de emparejar dos individuos una vez seleccionados.
  7. Elitism indica si hay que usar elitismo. Si es el caso, el mejor individuo de cada generacin se guarda para la generacin siguiente, previniendo su desaparicin.
  8. Norm normaliza los valores del fitness para evitar problemas deno linealidad de la funcin de fitness.
  9. Chrom.size longitud de cromosoma a utilizar.
  10. Population tamao de la poblacin.
  11. alfabeto a utilizar.
  12. Target fit. fitness objetivo para finalizar el proceso. Se recomienda definir el fitness para que el mximo target posible sea 1.
  13. Traceperiod cada cuntas generaciones actualiza la grfica y la informacin presentada.
  14. Load carga una poblacin previamente almacenada.
  15. Save graba la poblacin actual.
  16. Export exporta la poblacin actual en un formato de texto que pueda ser importado por otros programas.
  17. About muestra crditos del programa.
  18. Quit finaliza el programa.
  19. Help abre esta ayuda.
Información presentada¶
  1. Best fitness fitness del mejor individuo de la poblacin.
  2. Mean fitness fitness medio de la poblacin.
  3. Diversity diversidad de la poblacin.
  4. Epoch generaciones transcurridas en la evolucin en marcha.
  5. Bst fenotipo del mejor individuo en la generacin actual, es decir, mejor solucin al problema hasta el momento.

Procedimiento para generar una solución¶

  1. Seleccionar tipo de Algoritmo Gentico autilizar.
  2. Cargar la funcin de fitness con Load fitness.
  3. Definir el alfabeto a utilizar.
  4. Ajustar los diferentes parmetros (Pmut, Pemp, etc.) o dejar los valores por defecto.
  5. Click en Learn.
  6. El entrenamiento puede pararse a mano dando click en la opcin Stop o automticamente cuando el mejor individuo de la poblacin tenga un fitness igual o mayor a Target fit..
Interpretación de los valores de entrenamiento¶

Colores de las lneas

  1. Blanco: fitness medio de la poblacin.
  2. Verde: fitness del mejor individuo.
  3. Rojo: diversidad de la poblacin.
  4. Gris: subdivisiones del error en pasos del 10% del targetfitness, es decir, el punto en que se detendr la evolucin.

Ejemplo final¶

import genetic # para usar errot2fit

# pide el nmero de reinas
import tkinter.simpledialog
nr = tkinter.simpledialog.askstring('Parameter', 'How many queens?', initialvalue='32')
N = int(nr)

def count (ch): # cuenta el nmero de reinas mal colocadas
    mal = 0
    for i in range(len(ch)): # comprueba si est bien colocada
        for j in range(i+1,len(ch)):
            if ch[i]==ch[j]: # si se comen en horizontal (en vertical no pueden comerse por construccin)
                mal += 1
            elif abs(ch[i]-ch[j]) == abs(i-j): # si se comen en diagonal
                mal += 1
    return mal

def phenotype (ch):
    res = 'Unsolved: %d\n' % count(ch)
    res += f'{ch}'
    return res

def fitness (ch):
    # cuenta el nmero de reinas que cada una puede comerse
    return genetics.error2fit(count(ch), 10)
